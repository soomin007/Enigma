# ENIGMA Game — 개발 규칙

## 핵심 방향성

이 게임은 **암호 해독 퍼즐 게임**이다.
플레이어는 단서들을 조합해서 스스로 답을 추론해야 한다.
정보를 편하게 제공하는 것이 목표가 아니라, **추론의 과정 자체가 재미**여야 한다.

---

## 퍼즐 디자인 원칙

### 1. 단서는 단편적이어야 한다
각 단서 카드는 **전체 답의 한 조각**만 준다.
두 개 이상의 단서를 조합해야 비로소 설정값 하나가 나와야 한다.

- **금지**: 한 단서에서 "로터: I II III, 위치: AAA, 반사판: B" 모두 노출
- **권장**: 로터 순서는 사진에서, 초기 위치는 메모에서, 반사판은 심문에서 각각

### 2. 정보를 반복하지 않는다
같은 내용을 여러 단서에서 되풀이하면 추론의 여지가 사라진다.
단서들 사이에 **겹치는 정보가 없어야** 한다.

### 3. 힌트(hint_value)는 방향 제시이지 정답이 아니다
- **금지**: "아담의 첫 글자는 A. 세 로터 모두 초기 위치 A. 반사판은 B."
- **권장**: "리셋된 상태가 어떤 알파벳을 의미하는지 생각해보라."
- 힌트는 막힌 플레이어를 **다음 추론 단계로 안내**하는 용도다.

### 4. 해독기 초기 상태는 반드시 오답이어야 한다
플레이어가 해독기를 열었을 때 **쓰레기 값이 보여야 정상**이다.
초기 설정이 정답이면 해독기를 쓸 이유가 없다.

- Caesar 해독기: 슬라이더 0에서 시작
- Vigenère 해독기: 키 입력란 비어있음
- Substitution 해독기: 매핑 모두 비어있음
- Enigma 해독기: 위치 D/D/D, 반사판 A 등 **오답** 기본값

`cipher_params`는 서버 측 검증용 데이터다. 절대 UI 초기값으로 로드하지 않는다.

### 5. 설명 예시에 해당 챕터 정답을 사용하지 않는다
CIPHER_INTROS 오버레이나 UI 안내 문구의 예시 단어/값이
해당 챕터의 실제 정답(cipher_text, plain_text, 설정값)과 같으면 안 된다.

- **금지 예시**: 챕터 0(shift=3, ATTACK AT DAWN)에서 "예시: ATTACK → shift=3 → DWWDFN"
- **권장**: 관계없는 단어와 값 사용 (예: MARCH → shift=5 → RFWHM)

### 6. 간접 표현을 쓴다
단서는 1942년 실제 군사 문서·낙서·심문 형식이다.
답을 직접 말하지 않고, **맥락·행동·상황**으로 암시한다.

- **금지**: "반사판은 B입니다"
- **권장**: "우리 부대는 항상 매뉴얼 두 번째 반사판을 썼습니다"

### 7. 보고서 질문 선택지는 기본값 없음
OptionButton은 항상 `"── 선택 ──"` 플레이스홀더(인덱스 0)로 시작한다.
기본 선택이 정답인 경우는 절대 없어야 한다.

### 8. 레드 헤링(가짜 단서)을 넣는다
모든 단서가 정답을 향해 정렬되어 있으면 퍼즐이 아니라 안내서다.
챕터당 **의미 없는 단서 1~2개**를 섞어 플레이어가 스스로 걸러내게 한다.
예: 병사의 식단 불평 메모, 연애편지, 관계없는 날짜의 달력 사진.
가짜 단서는 그럴듯해 보여야 하지만 퍼즐 해결에는 기여하지 않는다.

### 9. 교차 검증(Cross-reference)을 요구한다
단서 A + 단서 B를 조합해야 단서 C의 의미가 밝혀지는 구조를 만든다.
어떤 하나의 단서만으로 설정값 하나를 완전히 확정할 수 없어야 이상적이다.
예: 심문 기록의 "일주일"(→ 숫자?) × 달력 사진의 "TAG 7"(→ 독일어?) → shift=7

## Godot 4 기술 규칙

### UI 노드 참조
이 프로젝트는 씬 파일(.tscn) 없이 GDScript에서 UI를 코드로 빌드한다.
생성된 노드는 반드시 **멤버 변수로 저장**해 참조한다.
인라인 `get_node("VBoxContainer/Panel/Label")` 같은 하드코딩 경로는 절대 사용하지 않는다.

```gdscript
# 금지
var text = get_node("VBoxContainer/Label").text

# 권장
var _my_label: Label  # 멤버 변수
_my_label = Label.new()
add_child(_my_label)
```

### 명시적 타입 선언
모든 변수와 함수 반환값에 타입 힌팅을 적용한다.
untyped Array/Dictionary를 인덱싱할 때는 `:=` 대신 명시적 타입 선언을 사용한다.

```gdscript
# 금지
var x := arr[i]

# 권장
var x: Dictionary = arr[i]
```

### 레드 헤링 오답 전용 피드백
플레이어가 레드 헤링(가짜 단서) 기반 오답을 제출했을 때는
"재검토하십시오" 같은 일반 메시지가 아닌,
해당 가짜 단서를 직접 저격하는 전용 텍스트를 출력한다.
`report_questions` 각 항목의 `red_herring_answers` 배열과 `red_herring_feedback` 문자열로 관리한다.
